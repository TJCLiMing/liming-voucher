# liming-voucher — 黎明教會付款申請單（代傳票）

真耶穌教會 黎明教會的付款申請單工具。前端填單 → 存進 Google Sheet → 列印 A4 代傳票 → 月底一鍵產生會計月報。

## 技術架構

| 層 | 技術 |
|---|---|
| 前端 | 單一 `index.html`（vanilla JS），部署到 GitHub Pages |
| 後端 | Google Apps Script Web App（`Code.gs`），提供 JSON API + 月報產生 |
| 資料 | Google Sheet（科目對照 + 申請單流水帳 + 會計月報分頁） |
| PDF | 瀏覽器列印 → 另存 A4 PDF |

前端與後端用 `text/plain` POST 溝通以避開 CORS preflight。

## 整體流程

```
申請人填單（GitHub Pages）
   → 存入 Google Sheet「申請單」流水帳
   → 列印 A4 代傳票（拾萬～元 位數格）
   → 月底出納：試算表選單「黎明教會 → 產生本月會計月報」
   → 自動生成「會計支出115MM」分頁（格式同會計 xlsx），直接下載交會計
```

## 設計定案（2026-07-05 討論）

- **科目**：前端下拉用自訂科目名稱（好記），「科目」分頁 B 欄放會計科目對照
  （如 `清掃薪資 → 6211薪資費用`）。產月報時自動映射；未對照的列標黃提醒。
- **一張單一個科目**（2026-07-05 補充定案）：科目在單頭選一次，明細列只有摘要＋金額；
  A4 預覽科目欄直向合併。對照會計 xlsx 的規則：一列＝一個科目，摘要彙總多筆。
- **流水號**：民國年月＋月內序，如 `11507-16`。一張單多列共用同一號；
  月報 NO 直接取「-」後的序號，與會計 xlsx 的月內編號一致。
- **金額位數格**：最高「拾萬」（上限 999,999），超過前端跳紅字。不加「百萬」欄。
- **會計月報格式**（比照會計 xlsx，B2 起）：
  `NO | 月 | 日期 | 科目代號 | 會計名稱 | 摘要 | 合計 | 領款人 | 備註`
  - 科目代號存「代號+名稱」完整字串（如 `6331伙食費`）
  - 會計名稱用 `=MID(E?,5,LEN(E?))` 公式切出
  - **同一張單（同流水號＋同科目）併成一列**：摘要用「、」串接，
    多筆金額寫成 `=a+b+c` 公式（比照會計現行習慣）
  - 末列 `=SUM(...)` 當月合計
- 會計原始 xlsx（`會計格式現支115年度*.xlsx`）**不進 git**（.gitignore 已排除）。

## Google Sheet 設定

- 分頁 **「科目」**：A 欄科目名稱、B 欄會計科目（代號+名稱）。
  可用試算表選單「黎明教會 → 初始化科目分頁」建立預設 11 個科目與對照。
- 分頁 **「申請單」**：不用手動建，第一次存檔自動生標題列。欄位：
  `流水號 | 序 | 日期 | 民國日期 | 科目 | 摘要 | 金額 | 領款人 | 申請人 | 合計 | 建立時間 | 備註`
  （備註欄給出納/會計註記，如「06/20已給會計」，會帶進月報）
- 分頁 **「會計支出115MM」**：由選單自動產生，重跑會整頁重建。

## GAS 部署

1. Google Sheet → 擴充功能 → Apps Script，貼上 `Code.gs`
2. 部署 → 新增部署作業 → 網頁應用程式
   - 執行身分：**我**
   - 誰可以存取：**任何人**
3. 複製 `/exec` 網址 → 填進 `index.html` 的 `GAS_API_URL`
4. 重新整理試算表後會出現「黎明教會」選單（產月報、初始化科目）

## 給 Claude Code 接手的重點

- 前端邏輯都在 `index.html` 的 `<script>`：`state` 是資料模型，`renderSheet()` 畫 A4 預覽，
  `splitDigits()` 金額拆位數，`COLS` 定義位數欄
- 後端 `Code.gs`：
  - `saveVoucher()` 一張單多列共用流水號；`buildNo_()` 掃當月（`11507-` 前綴）取最大序 +1
  - `buildMonthlyReport_(month)` 產月報：篩當月列 → 科目映射 → 寫入 B2 起版面 → MID/SUM 公式
  - `initSubjectSheet()` 建預設科目對照，已有資料不覆寫
- 沒填 `GAS_API_URL` 走本機示範模式（預設科目、存檔印 Console），方便改 UI
